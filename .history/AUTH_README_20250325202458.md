# MyLocalApp Authentication System

This document explains how to set up and configure the Supabase authentication system for MyLocalApp.

## Prerequisites

- A Supabase project (self-hosted or cloud)
- Node.js and npm installed

## Environment Setup

1. Rename `.env.example` to `.env` and update the values:

```
EXPO_PUBLIC_SUPABASE_URL=your_supabase_url
EXPO_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

## Database Setup

In your Supabase project, create the following tables:

### 1. Profiles Table

Create a `profiles` table with the following structure:

```sql
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  preferences TEXT[] DEFAULT '{}',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

### 2. Set up Row Level Security (RLS)

Enable Row Level Security on the profiles table and create the following policies:

```sql
-- Enable RLS
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;

-- Create policy for users to view only their own profile
CREATE POLICY "Users can view their own profile" 
  ON profiles 
  FOR SELECT 
  USING (auth.uid() = id);

-- Create policy for users to update only their own profile
CREATE POLICY "Users can update their own profile" 
  ON profiles 
  FOR UPDATE 
  USING (auth.uid() = id);

-- Create policy to allow insert of one's own profile
CREATE POLICY "Users can insert their own profile" 
  ON profiles 
  FOR INSERT 
  WITH CHECK (auth.uid() = id);
```

### 3. Set Up Database Functions

Create a function to automatically create a profile when a user signs up:

```sql
CREATE OR REPLACE FUNCTION public.handle_new_user() 
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.profiles (id, preferences)
  VALUES (new.id, '{}');
  RETURN new;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

### 4. Set Up Trigger

Create a trigger to call the function when a new user is created:

```sql
CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
```

## Authentication Flow

The app implements the following authentication flow:

1. On first launch, a "Willkommen" screen is shown with two options:
   - "Neu hier?" (New here?)
   - "Login mit existierendem Account" (Login with existing account)

2. New user flow:
   - User enters email
   - A temporary password is automatically generated
   - User selects preferences from categories: Kultur, Sport, Verkehr, Politik
   - An account is created and user is logged in

3. Login flow:
   - User enters email and password
   - User is authenticated and navigated to the main app

4. Profile management:
   - User can set/change password from the Profile tab
   - User can update preferences from the Profile tab
